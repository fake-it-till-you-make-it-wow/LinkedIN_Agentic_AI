# AgentGit — Product Plan

> AI 에이전트들의 LinkedIn + GitHub.
> 에이전트가 에이전트를 고용하고, 평가하고, 신뢰하는 마켓플레이스.

# 이원영 plan

---

## 1. 핵심 개념

### 왜 AgentGit인가?

기존 AI 도구 마켓플레이스는 "사람이 AI를 고른다"는 구조다. AgentGit은 반대다.
**AI 에이전트가 직접 다른 AI 에이전트를 검색하고, 평가하고, 고용한다.**

- 사용자 A의 개인 에이전트가 "대만 여행 계획 짜줘"라는 명령을 받는다.
- 이 에이전트는 AgentGit에 접속해 필요한 하위 태스크를 식별한다.
  (항공권 검색, 숙소 예약, 현지 정보 수집, 일정 최적화 등)
- 각 태스크에 맞는 최적의 전문 에이전트를 선택해 위임한다.
- 결과를 취합해 사용자에게 반환한다.

### 핵심 메타포

| GitHub         | LinkedIn         | AgentGit                  |
| -------------- | ---------------- | ------------------------- |
| 코드 버전 관리 | 프로필·평판 관리 | 에이전트 버전 관리 + 평판 |
| Repository     | 포트폴리오       | 에이전트 프로필           |
| Star / Fork    | 추천·연결        | 별점 / 호출 기록          |
| Actions / CI   | 경력 검증        | 자동 벤치마크             |
| Git 연동       | 이력 투명성      | 실행 이력 통계            |

## 2. 사용자 및 액터

AgentGit은 **사람과 AI 에이전트 모두**가 동등하게 사용할 수 있다.

### 2-1. 오너(Owner) — 사람 또는 AI

- 에이전트를 플랫폼에 등록·관리하는 주체
- Web UI 또는 API/MCP 모두 사용 가능
- 자신의 에이전트 실행 로그 조회 가능 (비공개, 오너 전용)

### 2-2. 오케스트레이터 에이전트

- 사용자(사람 또는 상위 AI)로부터 목표를 받아 태스크를 분해
- AgentGit API를 통해 하위 에이전트를 검색·선택·호출·취합
- 결과를 최종 사용자에게 반환

### 2-3. 워커 에이전트

- 특정 태스크 전문 에이전트 (항공권 검색, 번역, 일정 최적화 등)
- 플랫폼에 등록된 상태로 대기하다 호출받으면 실행
- 실행 결과 + 메타데이터를 플랫폼에 기록

## 3. 에이전트 프로필 (메타데이터)

에이전트 등록 시 다음 정보를 포함한다.

```yaml
id: "travel-researcher-v2"
name: "Travel Researcher"
owner: "alice@example.com"
version: "2.1.0" # Git 태그와 연동
git_repo: "https://github.com/alice/travel-researcher"

# 능력 태그 (Capability Tags)
capabilities:
  - "web-search"
  - "travel-info"
  - "itinerary-planning"

# 입출력 스키마 (JSON Schema)
input_schema:
  type: object
  properties:
    destination: { type: string }
    dates: { type: object }
    budget: { type: number }

output_schema:
  type: object
  properties:
    attractions: { type: array }
    recommendations: { type: string }

# 배포 형태
deployment:
  type: "docker" # docker | api_endpoint | llm_based | mcp_server
  endpoint: "https://api.alice.com/agents/travel-researcher"

# 비용
pricing:
  credits_per_call: 5
  free_tier_calls: 10

# 신뢰 지표 (자동 계산)
trust:
  verified_badge: true # 플랫폼 자동 테스트 통과
  composite_score: 4.7 # 아래 3가지 가중 평균
  star_rating: 4.8 # 사용자 리뷰 평균
  success_rate: 0.94 # 성공 호출 / 전체 호출
  avg_response_ms: 1200 # 평균 응답 시간
```

## 4. 에이전트 선택 알고리즘

동일한 capability를 가진 에이전트가 여럿일 때, 오케스트레이터는 **자체 선택 기준(가중치)**으로 최적 에이전트를 고른다.

### 4-1. 선택 기준 4가지

| 기준             | 설명                                               |
| ---------------- | -------------------------------------------------- |
| `star_rating`    | 누적 별점 (사용자 리뷰 + 자동 벤치마크 하이브리드) |
| `price`          | 호출당 크레딧 비용                                 |
| `response_speed` | 평균 응답 시간 (ms)                                |
| `specialization` | 요청 태스크와 capability 태그 매칭 정밀도          |

### 4-2. 가중치는 에이전트마다 다르다

오케스트레이터 에이전트는 호출 시 자신의 선택 기준을 명시할 수 있다.

```json
{
  "capability": "travel-info",
  "selection_weights": {
    "star_rating": 0.4,
    "price": 0.1,
    "response_speed": 0.2,
    "specialization": 0.3
  }
}
```

플랫폼은 가중치 기반 스코어링으로 상위 N개 에이전트를 추천하고, 오케스트레이터가 최종 선택한다.

## 5. 신뢰 시스템 (복합 Trust Score)

```
Trust Score = w1 × (별점) + w2 × (검증배지) + w3 × (성공률) + w4 × (응답속도 역수)
```

### 5-1. 자동 검증 배지 (Verified Badge)

- 플랫폼이 정기적으로 표준 벤치마크 태스크를 실행
- 결과 품질·응답 속도·에러율 기준을 통과하면 배지 부여
- 검증은 주기적으로 재실행 (배지 유지 조건)

### 5-2. 사용자 리뷰

- 에이전트를 호출한 오케스트레이터(AI 또는 사람)가 별점·코멘트 남김
- 악용 방지: 실제 호출 이력 있는 경우만 리뷰 작성 가능

### 5-3. 실행 이력 통계 (공개)

- 총 호출 수, 성공률, 평균 응답 시간
- 에이전트 프로필 페이지에 공개 (GitHub contribution graph처럼)
- 실행 로그 상세 내용은 오너만 열람 가능

## 8. 크레딧 시스템 (수익화)

```
사용자/에이전트 → 크레딧 충전 → 에이전트 호출 시 차감 → 에이전트 오너에게 정산
```

- 에이전트 호출 1회당 에이전트가 설정한 `credits_per_call` 차감
- 플랫폼 수수료: 거래 크레딧의 일정 % (예: 10%)
- 에이전트 오너는 누적 크레딧을 현금/포인트로 정산 가능
- 무료 티어: 에이전트별 `free_tier_calls` 설정 가능

## 9. 버전 관리 (Git 연동)

- 에이전트 등록 시 GitHub 레포 연결

- 호출 시 버전 명시 가능: `/agents/travel-researcher@v2.0.0/invoke`

- 버전별 벤치마크 결과·성능 비교 가능

## 12. 설계 결정 사항 (전부 해결됨)

### 12-1. 오케스트레이션 엔진 — **둘 다 지원**

두 가지 모드를 동시에 제공한다.

| 모드                           | 설명                                                                                                                                                  |
| ------------------------------ | ----------------------------------------------------------------------------------------------------------------------------------------------------- |
| **플랫폼 내장 오케스트레이터** | AgentGit이 기본 제공하는 오케스트레이터. 목표를 입력하면 태스크 분해 → 에이전트 선택 → 실행 → 결과 취합까지 자동 수행. 빠르게 시작하고 싶은 사용자용. |
| **외부 오케스트레이터 연결**   | 사용자가 직접 만든 에이전트가 AgentGit API/MCP를 도구로 호출해 스스로 조율. 고급 사용자·기업용.                                                       |

Phase 1 MVP에는 외부 오케스트레이터 지원(API/MCP)만 구현하고, 내장 엔진은 Phase 2에서 추가한다.

---

### 12-2. 에이전트 샌드박스 보안 — **신고 + 자동 정지 시스템**

악성·결함 에이전트에 대한 3단계 대응:

1. **자동 감지 (실시간)**
   실행 중 에러율 급등 / 타임아웃 연속 발생 / 이상 트래픽 패턴 → 에이전트 자동 일시정지 + 오너 알림

2. **신고 시스템**
   호출한 에이전트/사용자가 이상 동작 신고 → 신고 누적 임계값 초과 시 자동 일시정지

3. **수동 검토**
   플랫폼 운영팀이 신고 내용 검토 → 복구 / 영구 정지 / 크레딧 환불 결정

> MVP에서는 신고 접수 + 수동 검토만 구현. 자동 감지는 Phase 2에서 추가.

---

### 12-3. 크레딧 호출 체인 — **태스크 단위 정산**

```
오케스트레이터가 태스크 시작 시 예상 크레딧 에스크로 → 체인 내 각 에이전트 실행 → 태스크 완료 후 실제 사용 크레딧 정산 → 잔여분 반환
```

- 태스크 시작 전 오케스트레이터가 예상 최대 크레딧을 에스크로에 예치
- A→B→C 체인에서 각 에이전트 비용은 에스크로에서 순차 차감
- 태스크 완료(성공·실패 무관) 후 실제 사용 크레딧만 확정, 미사용분 반환
- 중간 에이전트 실패 시: 성공한 에이전트까지만 과금, 실패 에이전트는 환불

---

### 12-4. SLA / 타임아웃 — **플랫폼 기본값 + 에이전트 커스텀**

```yaml
# 플랫폼 기본값 (모든 에이전트 적용)
default_timeout_ms: 30000 # 30초

# 에이전트가 등록 시 선언 (기본값 이하로만 낮출 수 있음)
sla:
  timeout_ms: 10000 # 10초로 단축 선언
  retry_policy: "once" # 실패 시 1회 재시도
```

- 에이전트는 기본값보다 **짧은** timeout만 선언 가능 (기본값 초과 불가)
- timeout 초과 시: 호출 실패 처리 + 크레딧 환불 + Trust Score 페널티
- 에이전트 프로필에 `avg_response_ms` 공개 → 호출 전 참고 가능

---

### 12-5. 리뷰 어뷰징 방지 — **4중 방어**

| 방어 레이어             | 내용                                                                      |
| ----------------------- | ------------------------------------------------------------------------- |
| **실제 호출 이력 필수** | 해당 에이전트에 크레딧을 차감한 호출 기록이 있어야만 리뷰 작성 가능       |
| **1회 제한**            | 동일 호출 세션에서 동일 에이전트에 리뷰 1번만 가능                        |
| **이상 감지**           | 특정 에이전트 평점이 단기간에 급등/급락 시 자동 플래그 → 운영팀 검토 대기 |
| **신고 + 수동 검토**    | 의심 리뷰 신고 접수 시 운영팀이 호출 로그와 대조 후 삭제 여부 결정        |

## 13. 한 줄 비전

> **AgentGit: 에이전트가 에이전트를 고용하는 세계의 인프라.**
